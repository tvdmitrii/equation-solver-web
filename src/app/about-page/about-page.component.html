Overview

This is a proof of concept iterative equation solver which uses Broyden's method to find roots for sets of implicit equations, resulting in a powerful tool for modeling physical systems. The key feature of the solver is the use of, what I call, Equation Modeling Language which has a C++-like grammar but uses equations as the main building block, instead of simple assignments.

This is not a Wolfram|Alpha alternative. For example, it will not find both roots of a quadratic equation, or it will not solve the equations symbolically. Equation Modelling Language is designed to give a convenient way to model engineering/physical systems which typically do not have multiple solutions. You can shift it to a specific solution by adjusting the guess values in the "Guess Value" tab.

Example. Block on an Inclined Plane.

Consider a classical 2D physics problem, where one has a block of mass M on an inclined plane in a gravitational field with vector g pointing down. Friction is holding the block stationary. At a certain angle α between the inclined plane and the horizontal plane, the block will begin sliding. What is that angle, assuming that the friction coefficient μ is known?

Easy! Just write down the governing equations! In this case, they are mainly Newton's second law equations along the inclined plane and perpendicular to the inclined plane:

N - M * g * cos(alpha) = 0
F - M * g * sin(alpha) = 0
F = mu * N

Eventually, one would also be able to conduct parametric studies to see how various parameters affect the result. Here, one would find that M and g make no difference. Of course, for such a simple system you can simplify the equations to tan(α) = μ. However, it would be less obvious for a much more complicated system where, for example, some variables come from thermodynamic property lookup tables.

I believe this could be a powerful learning tool for both high-school students and research scientists.
